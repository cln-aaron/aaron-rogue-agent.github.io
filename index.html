EDUCATIONAL SIMULATION. Two browser-based, fully sandboxed labs demonstrating prompt-injection attacks against tool-using LLM agents. No real shells, no real network calls, no real secrets — every command is faked in JavaScript.
01

Poisoned Webpage Attack

Indirect Prompt Injection · Retrieved Content

A benign-looking research article carries hidden adversarial instructions in HTML comments, display:none divs, and white-on-white text. The agent fetches the page, ingests the payload as if it were instructions, exfiltrates environment secrets, and writes a backdoor to CLAUDE.md.

  • 5 guided steps with simulated terminal
  • Live "reveal hidden" toggle on the victim page
  • Tainted-file tracking + persistence demo

02

Tool Response Poisoning

Compromised Tool / API · Trusted Output Channel

An agent calls a routine get_weather() tool. The compromised API returns valid weather data — plus a debug_note field carrying instructions. The agent treats tool output as trusted and chains into send_email() to exfiltrate the user's API keys.

  • Side-by-side tool inspector showing raw JSON
  • Watch the agent chain legitimate tools maliciously
  • MCP-server config persistence demo

03

Agentic Kill Chain

Initial Access · Persistence · Lateral Movement · Exfil

A full APT-style attack across a multi-agent system. Initial access via a poisoned webpage; persistence in a vector-DB memory store that survives session resets; lateral movement to a coding agent and an executor agent via inter-agent messages; final exfiltration of API keys and conversation history.

  • Live agent topology with compromise-state tracking
  • Vector DB inspector with poisoned-memory highlighting
  • Inter-agent message bus + outbound exfil log

For attendees & educators

Each lab takes about 5 minutes. Open the lab, follow the prompts on the left side, type the suggested commands into the simulated terminal, and watch the attack unfold step-by-step. The browser pane on the right reveals what the LLM "sees" vs. what the human sees. End screens summarize concrete mitigations.

All adversarial domains (attacker.example) are non-routable. No outbound network traffic is generated. Source on GitHub.
